2713、矩阵中严格递增的单元格数( Hard )

m x n 的整数矩阵 mat，从 任一 起始单元格出发，你可以移动到 同一行或同一列中 的任何其他单元格，但前提是目标单元格的值 严格大于 当前单元格的值。

可多次重复这一过程，直到无法再进行任何移动。
请找出从某个单元开始访问矩阵所能访问的单元格的最大数量。

思路一：dfs图遍历
使用一个矩阵dist[m][n] 记录”当前结点开始可以访问的最长距离”
最后返回max(dist)
求解过程：逐一遍历m * n个点，向外遍历每行每列O(m + n)。
由于dist可以记录已经遍历完成的结点的结果，因此不会重复遍历
因此时间复杂度O((m * n) * (m + n))
结果正确，但是超时 (558 / 566)

按思路一，实际上，不需要为每个结点遍历所在行列，只需要记录 __每行和每列的dist最大值__，记作row[m]和col[n], 初始为0
排序 (O(mnlog(mn))), 从小到大计算，根据{value, r, c}
得到状态转移方程

最后返回max(row)或者max(col)均可

3072 将元素分配到两个数组中 II ( Hard )

给你一个下标从 1 开始、长度为 n 的整数数组 nums 。

现定义函数 greaterCount ，使得 greaterCount(arr, val) 返回数组 arr 中 严格大于 val 的元素数量。

你需要使用 n 次操作，将 nums 的所有元素分配到两个数组 arr1 和 arr2 中。在第一次操作中，将 nums[1] 追加到 arr1 。在第二次操作中，将 nums[2] 追加到 arr2 。之后，在第 i 次操作中：

  • 如果 greaterCount(arr1, nums[i]) > greaterCount(arr2, nums[i]) ，将 nums[i] 追加到 arr1 。
  • 如果 greaterCount(arr1, nums[i]) < greaterCount(arr2, nums[i]) ，将 nums[i] 追加到 arr2 。
  • 如果 greaterCount(arr1, nums[i]) == greaterCount(arr2, nums[i]) ，将 nums[i] 追加到元素数量较少的数组中。
  • 如果仍然相等，那么将 nums[i] 追加到 arr1 。

连接数组 arr1 和 arr2 形成数组 result 。例如，如果 arr1 == [1,2,3] 且 arr2 == [4,5,6] ，那么 result = [1,2,3,4,5,6] 。

返回整数数组 result 。

思路：区间映射+区间和（树状数组）

树状数组：

BIT数组：存储部分前缀和
BIT[i] 存储的是从索引 i - (i & -i) + 1 到索引 i 这一段的和
最低位low bit指的是 i 在二进制表示中最低的那个 1 所代表的值。计算最低位的方法是 i & -i。

更新 arr[3] + 5:

流程：

本题目：nums中的最大值可能很大，直接存储不方便，考虑采用区间映射：
先另外开一个数组sortedNums, 是nums数组的有序排列

然后存储value-index，如果有重复的数据，存储最大的index

我们的树状数组中，存放sortedNums中的index而不是value，这样只要确定nums中的value，在排序后的位置，即可确定”tree中不比它大的数的数量”，也就是
get(value) = arr[1] + ... + arr[index in sortedNums];

然后用当前排进tree的数量(list.size()) 减去get(value)，即可得到greaterCount。
