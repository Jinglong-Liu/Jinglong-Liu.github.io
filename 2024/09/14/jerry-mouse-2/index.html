Servlet 简介

首先，什么是Servlet ? 询问chatgpt，得到回答：

Servlet 是 Java EE（现为 Jakarta EE）平台上的一种服务器端组件，用于处理客户端请求并生成动态响应。它通常用于构建基于 HTTP 协议的 Web 应用程序。

Servlet 的核心功能：

  • 处理 HTTP 请求：Servlet 接收来自客户端（通常是浏览器）的 HTTP 请求，例如 GET、POST 请求。
  • 生成 HTTP 响应：Servlet 处理请求后，生成相应的 HTTP 响应，例如返回 HTML 页面、JSON 数据等。
  • 生命周期管理：Servlet 由 Servlet 容器（如 Tomcat、Jetty）管理，其生命周期包括初始化（init）、处理请求（service）和销毁（destroy）等阶段。

Servlet 的应用场景：

  • Web 应用中的动态内容生成。
  • RESTful API 的实现。
  • 处理表单数据、文件上传等。

Servlet 是 Java Web 开发的基础，常与 JSP、Spring MVC 等技术结合使用。

Tomcat是所谓的Servlet容器，因此，这个servlet，或者app，是要被放到Tomcat或类似的容器中运行的。这些接口的实现类也都是Tomcat提供的。因此，这也是本项目的主要内容–编写一个jerry-mouse-server，来当作servlet的容器，替代掉Tomcat

写回数据

根据测试用例，本次必须实现的接口

我们回到前一章，检查之前是如何写回数据的

现在则要求在doGet, doPost等方法中，通过response的writer和outputStream两个接口实现

因此可以在Response的实现类中实现。

根据规范，writer和outPutStream在一个方法中，只能使用一个。因此不必考虑同时使用两个方法的情况，这降低了维护的难度。

另外需要注意，两种方式，均可以通过多次write, 一次flush的方式一次性写入，因此，在write的适合，实际是需要
维护一个缓冲区，而在flush的时候真正写入, 也就是写入Channel。

下面我们分别讨论两个接口的实现。

writer

getWriter() 返回 PrintWriter 类型，因此我们的实现，需要是PrintWriter的子类。

查看PrintWriter 源码，发现其在构造方法引入一个内部的Writer, 然后write实际上通过内部Writer执行，维护缓冲区，而flush的时候写入。因此，我们可以自己构建一个StringWriter，作为PrintWriter内部的Writer, 并重写PrintWriter的flush方法即可。

而真正写入channel，需要调用ReadChannelHandler相关接口，在构造方法传入即可。

outputStream

通过outputStream写入数据，其最终仍然调用ReadChannelHandler(socketWriter接口)的方法。

通过Stream的方式，其实现类似。内部维护一个ByteArrayOutputStream缓冲，在flush的时候真正写入
