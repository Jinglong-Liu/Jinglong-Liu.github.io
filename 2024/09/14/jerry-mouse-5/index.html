cookie, session简介

Cookie 和 Session 是 Web 开发中常用的状态管理机制，用于保持用户状态或在多个请求之间共享数据。

两者一般结合使用

session使用原理

  • 客户端请求：客户端第一次访问服务器时，服务器创建一个 Session 并生成一个唯一的 Session ID。
  • 生成 Session ID：服务器将 Session ID 返回给客户端，客户端通常将 Session ID 存储在 Cookie 中。
  • 后续请求：客户端的后续请求会带上 Session ID，服务器根据 Session ID 找到对应的会话数据，从而保持用户状态。
  • Session 失效：当用户会话结束或超时，服务器可以删除该 Session，释放资源

session的作用

  • 用户身份验证：通过 Session 保存用户的登录状态，使得用户在浏览多个页面时无需重复登录。
  • 购物车：在电子商务网站中，可以使用 Session 保存用户的购物车数据，直到结账完成。
  • 临时数据存储：用于保存用户在短期内需要的临时信息，如表单的中间状态、搜索条件等。
