Spring Boot 工作方式

在支持spring boot前，先弄清楚其工作方式。

首先让chatgpt简单介绍一下Spring Boot(以下内容来自chatgpt)

Spring Boot 是基于 Spring 框架的开源框架，旨在简化 Spring 应用开发。它提供了一种快速构建生产级 Spring 应用的方式，尤其是在微服务架构下非常受欢迎。以下是 Spring Boot 的一些主要特点和优势：

    1. 自动配置
      Spring Boot 提供自动配置功能，可以根据应用的依赖自动配置 Spring 的 Bean，减少了手动配置的麻烦。例如，当你引入 spring-boot-starter-web 依赖时，Spring Boot 会自动配置一个 Web 应用环境，无需手动编写大量配置文件。
    1. 嵌入式服务器
      Spring Boot 提供了嵌入式的 Tomcat、Jetty、Undertow 等服务器，可以直接在项目中运行 Web 应用，无需额外配置外部服务器。你可以直接通过 java -jar 命令运行打包后的 Spring Boot 应用。
    1. 简化的依赖管理
      Spring Boot 提供了大量的 “starter” 依赖包（如 spring-boot-starter-web、spring-boot-starter-data-jpa 等），这些包预先定义了常用的依赖组合，开发者可以通过引入这些 starter 简化依赖管理。
    1. 健康检查与监控
      通过 Spring Boot Actuator，开发者可以方便地监控应用的运行状态，包括健康检查、度量数据、内存信息等。
    1. 生产就绪特性
      Spring Boot 为应用提供了许多生产级别的特性，包括日志、异常处理、健康检查、应用监控等，方便开发者快速构建稳定、可靠的应用。
    1. 外部化配置
      Spring Boot 支持通过 application.properties 或 application.yml 文件配置应用，也可以从外部源（如环境变量、命令行参数）动态加载配置，支持不同环境的配置管理。
    1. 微服务友好
      Spring Boot 是 Spring Cloud 的基础，它简化了微服务开发，集成了许多微服务架构中常用的功能，如服务注册与发现、负载均衡、配置管理等。

作为入门，本章节仅研究与web相关的内容。主要是 嵌入式服务器 。当你引入spring-boot-starter-web 时，会引入一个嵌入式的Tomcat。当你启动SpringApplication.run时，程序不会立即结束，而是监听一个端口。实际上，就是启动了内置的Tomcat。

内置服务器是可以定制的，SpringBoot 提供了三种服务器，当然，也可以定制成我们自己写的jerry-mouse，但是，需要我们自己编写相应的Factory，适配相关的接口。
