  • 认证授权
  • 日志记录和监控
  • 参数过滤

FilterConfig 和 FilterChain

config用来存在param等信息，在Filter执行init()时当作参数使用。chain来管理Filter的执行顺序

我们的执行流程需要调整：匹配到servlet后，不再直接执行httpServlet.service(request, response);, 而是要先通过若干Filter, 而这由FilterChain管理。因此，一种可能是实现：

Filter可以执行匹配的Servlet，指定某些Servlet执行这个Filter。因此需要对Filter进行管理。

一种比较清晰的管理方式是，ApplicationContext(ServletContext实现类，非Spring的ApplicationContext)中，保管这些manager, 遇到相关操作，发配给各种manager操作。

Filter生效的流程：

  • 从web.xml中，读取Filter, 并加载到manager统一管理

  • 根据请求url，筛选出应当生效的Filter, 由FilterChain负责调度

  • 调度顺序为：filter 1 -> ... -> filter n -> service(req, resp) -> filter n -> ... -> filter 1

前两部，仿照之前解析处理Servlet的过程，容易实现，下面问题在于如何正确调度这些Filter的顺序，也即FilterChain的实现类的doFilter的实现

Listener

Listener 是 Java EE 规范中的一种组件，它用于监听 Web 应用程序中的各种事件。Listener 可以监听以下几类事件：

  • ServletContext 生命周期事件：当 ServletContext 被创建或销毁时触发。
  • Session 生命周期事件：当一个用户会话（HttpSession）被创建、销毁或属性发生变化时触发。
  • 请求事件：当请求进入 Web 应用程序或请求的属性发生变化时触发。

Listener种类

罗列一下，一共有这么几种Listener

ServletContext 相关的 Listener

  • ServletContextListener：监听 Web 应用的启动和关闭。

  • ServletContextAttributeListener：监听 ServletContext 中属性的增加、删除或替换。

HttpSession 相关的 Listener

  • HttpSessionListener 用于监听会话的创建和销毁。

  • HttpSessionAttributeListener 用于监听 HttpSession 中的属性添加、删除或修改。

  • HttpSessionBindingListener：当对象绑定到 HttpSession 时（如使用 setAttribute），对象本身实现此接口，从而感知自身被绑定或解绑。

  • HttpSessionIdListener：监听 HttpSession 的 ID 变化

ServletRequest 相关的 Listener

  • ServletRequestListener：监听 ServletRequest 的创建和销毁。

  • ServletRequestAttributeListener：监听 ServletRequest 中属性的增加、删除或替换。
